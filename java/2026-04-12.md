### 1. `StampedLock`으로 낙관적 읽기(Optimistic Read) 패턴 구현하기

`ReentrantReadWriteLock`은 읽기 락도 공유 락을 획득해야 하지만, Java 8에서 도입된 `StampedLock`은 **낙관적 읽기(tryOptimisticRead)** 모드를 제공해 락 획득 없이 읽기를 시도하고, 이후 쓰기가 없었는지 `validate()`로 검증하는 패턴을 지원합니다. 읽기가 압도적으로 많은 시나리오에서 쓰기 락 경합 없이 성능을 크게 높일 수 있습니다.

```java
public class Point {
    private final StampedLock lock = new StampedLock();
    private double x, y;

    public double distanceFromOrigin() {
        long stamp = lock.tryOptimisticRead();   // 락 없이 스탬프 발급
        double curX = x, curY = y;

        if (!lock.validate(stamp)) {             // 읽는 사이 쓰기가 발생했으면
            stamp = lock.readLock();             // 일반 읽기 락으로 폴백
            try {
                curX = x; curY = y;
            } finally {
                lock.unlockRead(stamp);
            }
        }
        return Math.hypot(curX, curY);
    }

    public void move(double deltaX, double deltaY) {
        long stamp = lock.writeLock();
        try { x += deltaX; y += deltaY; }
        finally { lock.unlockWrite(stamp); }
    }
}
```

> 💡 **왜 중요한가**: 읽기 빈도가 높은 캐시·좌표·설정 객체에서 `synchronized`나 `ReadWriteLock` 대비 불필요한 블로킹을 없애 스루풋을 수 배 향상시킬 수 있다.

### 2. `Map`의 함수형 메서드로 조건부 업데이트 간결하게 처리하기 (`computeIfAbsent`, `merge`)

Java 8부터 `Map`에 추가된 `computeIfAbsent`, `merge`, `compute`, `replaceAll` 등은 "없으면 초기화 후 추가, 있으면 갱신"이라는 패턴을 `if`문 없이 단일 호출로 표현하게 해줍니다. `merge`는 특히 값이 없을 때와 있을 때를 하나의 람다로 통합할 수 있어 집계·카운팅 코드를 크게 단순화합니다.

```java
// ❌ 전통 방식 — null 체크 반복
Map<String, List<String>> groups = new HashMap<>();
if (!groups.containsKey(key)) groups.put(key, new ArrayList<>());
groups.get(key).add(value);

// ✅ computeIfAbsent — 키 없을 때만 초기화
groups.computeIfAbsent(key, k -> new ArrayList<>()).add(value);

// ✅ merge — 단어 빈도 카운팅
Map<String, Integer> freq = new HashMap<>();
words.forEach(w -> freq.merge(w, 1, Integer::sum));
// 없으면 1로 초기화, 있으면 기존 값 + 1

// ✅ compute — 읽기·수정을 원자적으로 (null 처리 포함)
freq.compute("java", (k, v) -> v == null ? 1 : v * 2);
```

> 💡 **왜 중요한가**: 멀티맵, 빈도 집계, 캐시 초기화 등 실무에서 반복되는 패턴을 null-safe하고 가독성 높은 한 줄로 줄여주며, `ConcurrentHashMap`에서도 원자성을 보장한다.
