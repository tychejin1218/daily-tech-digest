# IT 뉴스 - 2026-04-08

### OpenAI, Responses API에 '내장 웹 검색' 도구 정식 GA — RAG 파이프라인 간소화

OpenAI는 Responses API의 `web_search_preview` 도구를 정식 GA로 전환하며, gpt-4o 및 gpt-4o-mini에서 추가 인프라 없이 실시간 웹 검색 결과를 함께 반환할 수 있게 됐다. 기존에는 별도 검색 API 연동·청킹·임베딩 저장소를 직접 구성해야 했으나, 이제 단일 API 호출로 최신 정보가 포함된 응답을 얻을 수 있다. 파일 검색·코드 실행 등 다른 내장 도구와 병렬 호출도 지원돼 에이전틱 워크플로 구성이 한층 단순해졌다.

> 💡 **왜 중요한가**: RAG 아키텍처의 복잡도를 API 한 줄로 대체할 수 있어, AI 백엔드 개발 비용과 유지보수 부담이 크게 줄어든다.

### AWS Lambda, 런타임 외부 확장 없이 최대 15분→60분 실행 시간 한도 상향 발표

AWS는 Lambda 함수의 최대 실행 시간을 기존 15분에서 60분으로 늘리는 변경을 발표했다. 이는 LLM 추론·배치 임베딩 생성·대용량 파일 처리처럼 장기 실행이 필요한 서버리스 워크로드를 별도 EC2 인스턴스나 Fargate 없이 처리할 수 있게 해 준다. 추가 비용 구조는 기존 GB-초 요금제와 동일하며, 한도 상향은 Graviton3 런타임부터 우선 적용된다.

> 💡 **왜 중요한가**: AI 파이프라인의 서버리스 전환 장벽이 낮아지며, 배치성 LLM 작업을 상시 서버 없이 비용 효율적으로 운영할 수 있게 된다.

### Cursor 0.50, 멀티-레포 컨텍스트 인덱싱 및 MCP 서버 자동 연결 지원

AI 코드 에디터 Cursor가 0.50 버전을 출시하며 여러 저장소를 동시에 인덱싱해 크로스-레포 참조를 에디터 내에서 바로 처리하는 기능을 추가했다. 또한 MCP(Model Context Protocol) 서버를 프로젝트 루트의 `.cursor/mcp.json`에 선언하면 팀원 전체가 동일한 컨텍스트 소스(DB 스키마, API 문서 등)를 자동으로 공유할 수 있다. Background Agent 기능도 개선돼 장시간 리팩터링 작업을 에디터를 닫아도 클라우드에서 이어서 실행할 수 있다.

> 💡 **왜 중요한가**: AI 코딩 어시스턴트가 단일 파일 자동완성을 넘어 팀 단위 컨텍스트 공유 플랫폼으로 진화하며, 대규모 코드베이스에서의 실효성이 크게 높아진다.
