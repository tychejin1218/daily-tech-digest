### 1. `switch` 패턴 매칭 (Java 21) — 타입 분기의 새로운 표준

Java 21부터 `switch` 표현식에서 타입 패턴과 가드 조건(`when`)을 직접 사용할 수 있어, `instanceof` + 캐스팅 체인을 대체한다. `null` 처리도 `switch` 안에서 명시적으로 다룰 수 있어 NPE 위험이 줄어든다.

```java
sealed interface Shape permits Circle, Rectangle, Triangle {}

String describe(Shape shape) {
    return switch (shape) {
        case Circle c when c.radius() > 100 -> "대형 원: " + c.radius();
        case Circle c                        -> "원: " + c.radius();
        case Rectangle r when r.width() == r.height() -> "정사각형";
        case Rectangle r                     -> "직사각형 %dx%d".formatted(r.width(), r.height());
        case Triangle t                      -> "삼각형";
    };
}
```

`sealed` 인터페이스와 함께 쓰면 컴파일러가 모든 케이스를 커버했는지 검사해줘서 `default` 없이도 완전성(exhaustiveness)을 보장한다.

> 💡 **왜 중요한가**: 복잡한 타입 분기 로직을 선언적으로 표현할 수 있어 코드 가독성과 안전성이 동시에 올라간다.

### 2. Structured Concurrency (구조적 동시성, Java 21 Preview → Java 25 정식 예정)

`StructuredTaskScope`를 사용하면 여러 비동기 작업을 하나의 범위(scope) 안에서 생명주기를 묶어 관리할 수 있다. 자식 스레드 중 하나가 실패하거나 완료되면 나머지를 자동으로 취소해주므로 리소스 누수와 복잡한 예외 처리가 사라진다.

```java
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Subtask<User>  user  = scope.fork(() -> fetchUser(id));
    Subtask<Order> order = scope.fork(() -> fetchOrder(id));

    scope.join().throwIfFailed();   // 둘 다 성공할 때까지 대기, 실패 시 예외 전파

    return new Response(user.get(), order.get());
}
// scope 종료 시 살아있는 자식 스레드 자동 취소
```

`ShutdownOnSuccess`를 사용하면 "가장 빠른 응답 하나만 받고 나머지는 취소"하는 패턴도 간단하게 구현된다. Virtual Thread와 결합하면 스레드 비용 걱정 없이 고수준 동시성 제어가 가능하다.

> 💡 **왜 중요한가**: 기존 `ExecutorService`의 누수 취약점과 복잡한 취소 로직을 구조적으로 해결해, 동시성 코드를 동기 코드만큼 직관적으로 읽고 추론할 수 있게 한다.
