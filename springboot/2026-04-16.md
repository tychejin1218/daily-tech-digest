### 1. `HttpInterface`로 선언적 HTTP 클라이언트 작성 (Spring 6+)

Spring 6부터 도입된 `@HttpExchange` 기반 HTTP Interface를 사용하면, Feign 없이도 인터페이스 선언만으로 외부 API 클라이언트를 구성할 수 있습니다. `RestClient`나 `WebClient`를 프록시 팩토리에 연결하면 구현체가 자동 생성되어 보일러플레이트 코드가 대폭 줄어듭니다.

```java
@HttpExchange(url = "/api/users", accept = "application/json")
public interface UserClient {

    @GetExchange("/{id}")
    UserDto getUser(@PathVariable Long id);

    @PostExchange
    UserDto createUser(@RequestBody CreateUserRequest request);
}

@Configuration
public class ClientConfig {
    @Bean
    UserClient userClient(RestClient.Builder builder) {
        RestClient restClient = builder.baseUrl("https://external-api.example.com").build();
        return HttpServiceProxyFactory
                .builderFor(RestClientAdapter.create(restClient))
                .build()
                .createClient(UserClient.class);
    }
}
```

> 💡 **왜 중요한가**: 외부 서드파티 라이브러리(OpenFeign) 의존 없이 Spring 네이티브만으로 타입 안전한 선언적 HTTP 클라이언트를 구현할 수 있어 유지보수성과 테스트 용이성이 크게 향상됩니다.

### 2. GraalVM Native Image로 Spring Boot 앱 기동 시간·메모리 최적화

Spring Boot 3.x는 GraalVM Native Image 빌드를 공식 지원하며, AOT(Ahead-of-Time) 컴파일을 통해 애플리케이션 시작 시간을 수십 밀리초 수준으로, 메모리 사용량을 기존 JVM 대비 절반 이하로 줄일 수 있습니다. 특히 서버리스(Lambda, Cloud Run) 환경에서 콜드 스타트 문제를 근본적으로 해결합니다.

```bash
# Maven으로 네이티브 이미지 빌드
./mvnw -Pnative native:compile

# Gradle로 네이티브 이미지 빌드
./gradlew nativeCompile
```

```java
// 리플렉션 사용 시 힌트 등록 필요
@RegisterReflectionForBinding({UserDto.class, OrderDto.class})
@SpringBootApplication
public class MyApplication { }

// 런타임 힌트를 직접 등록하는 방식
public class MyRuntimeHints implements RuntimeHintsRegistrar {
    @Override
    public void registerHints(RuntimeHints hints, ClassLoader classLoader) {
        hints.reflection().registerType(MyDynamicClass.class,
            MemberCategory.INVOKE_PUBLIC_METHODS);
    }
}
```

> 💡 **왜 중요한가**: 컨테이너·서버리스 환경에서 콜드 스타트와 메모리 비용은 곧 운영 비용이며, 네이티브 이미지는 이를 획기적으로 절감하는 현시점 가장 실용적인 방법입니다.
