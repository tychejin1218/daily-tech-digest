### 1. Unnamed Variables (`_`)로 의도를 명확하게 표현하기 (Java 22 정식)

Java 22부터 사용하지 않는 변수나 패턴을 `_`(underscore)로 명시적으로 표기할 수 있습니다. `catch` 블록의 예외 객체, 람다의 미사용 파라미터, `for` 루프의 인덱스 등에서 "의도적으로 무시함"을 코드에 직접 표현할 수 있어 가독성과 컴파일러 경고를 동시에 해소합니다.

```java
// catch에서 예외 내용 불필요할 때
try {
    int value = Integer.parseInt(input);
} catch (NumberFormatException _) {
    System.out.println("숫자가 아닙니다");
}

// 람다에서 두 번째 파라미터가 필요 없을 때
map.forEach((key, _) -> System.out.println(key));

// switch 패턴에서 특정 타입만 체크할 때
switch (obj) {
    case Integer _ -> System.out.println("정수입니다");
    case String s  -> System.out.println("문자열: " + s);
}
```

> 💡 **왜 중요한가**: 변수명 `ignored`, `unused`, `e` 같은 관례적 네이밍 대신 언어 차원에서 "무시"를 표현해 팀 전체의 코드 의도가 통일됩니다.

### 2. `Collectors.teeing()`으로 스트림 단일 순회로 두 가지 집계 동시에 처리하기 (Java 12+)

`Collectors.teeing()`은 하나의 스트림을 두 개의 하위 컬렉터(downstream collector)로 동시에 보내고, 두 결과를 `BiFunction`으로 합쳐 반환합니다. 평균과 합계, 최솟값과 최댓값처럼 두 가지 집계가 필요할 때 스트림을 두 번 순회하지 않아도 됩니다.

```java
record Stats(double avg, int sum) {}

Stats stats = IntStream.rangeClosed(1, 10)
    .boxed()
    .collect(Collectors.teeing(
        Collectors.averagingInt(Integer::intValue),  // 첫 번째 집계: 평균
        Collectors.summingInt(Integer::intValue),    // 두 번째 집계: 합계
        Stats::new                                   // 두 결과를 합치는 함수
    ));

System.out.println(stats); // Stats[avg=5.5, sum=55]

// 최솟값/최댓값 동시 추출
var range = list.stream().collect(Collectors.teeing(
    Collectors.minBy(Comparator.naturalOrder()),
    Collectors.maxBy(Comparator.naturalOrder()),
    (min, max) -> Map.of("min", min.orElseThrow(), "max", max.orElseThrow())
));
```

> 💡 **왜 중요한가**: 대용량 스트림에서 집계 항목이 늘어날수록 순회 횟수가 성능 병목이 되는데, `teeing()`은 단일 패스로 두 집계를 처리해 I/O·메모리 효율을 함께 잡습니다.
