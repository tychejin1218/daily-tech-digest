### 1. Scoped Values로 ThreadLocal을 대체하여 Virtual Thread 환경에서 안전하게 컨텍스트 공유하기 (Java 25 정식)

`ScopedValue`는 불변(immutable)이고 명확한 생명주기를 가지는 컨텍스트 전달 메커니즘으로, 수백만 개의 Virtual Thread를 사용할 때 `ThreadLocal`이 가지는 메모리 누수와 상속 비용 문제를 해결합니다. `ScopedValue.where(USER, currentUser).run(() -> service.process())` 형태로 호출 범위를 명시적으로 한정하며, 바인딩이 끝나면 자동으로 해제되어 GC 부담이 없습니다. 또한 불변이기 때문에 자식 스레드(`StructuredTaskScope`)에 안전하게 상속할 수 있고, JIT가 상수처럼 최적화할 수 있어 `ThreadLocal.get()`보다 빠른 경우가 많습니다.

> 💡 **왜 중요한가**: Virtual Thread 시대에는 스레드당 데이터 격리 비용이 누적되므로, 불변·범위 한정 컨텍스트가 정답이다.

### 2. JEP 483 Ahead-of-Time Class Loading & Linking으로 JVM 시작 시간 단축하기 (Java 24 정식)

`-XX:AOTMode=record`로 트레이닝 런을 한 번 실행한 뒤 `-XX:AOTMode=create`로 AOT 캐시를 생성하면, 이후 실행 시 클래스 로딩과 링킹 단계가 건너뛰어져 Spring Boot 같은 대형 애플리케이션의 시작 시간이 30~40%까지 줄어듭니다. 기존 CDS(Class Data Sharing)를 확장한 것이지만 별도 빌드 도구나 코드 변경 없이 JVM 옵션만으로 적용할 수 있고, GraalVM Native Image와 달리 동적 기능(리플렉션, 동적 클래스 로딩)을 그대로 유지합니다. 서버리스 환경의 콜드 스타트나 CLI 도구처럼 시작 시간이 중요한 워크로드에 즉시 효과를 볼 수 있습니다.

> 💡 **왜 중요한가**: 코드 한 줄 안 바꾸고 JVM 플래그만으로 콜드 스타트를 단축할 수 있는 가장 가성비 좋은 최적화다.
