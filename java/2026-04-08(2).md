### Record 클래스로 불변 데이터 객체 간결하게 선언하기

Java 16에서 정식 도입된 `record`는 불변 데이터 캐리어 클래스를 선언할 때 생성자, getter, `equals`, `hashCode`, `toString`을 자동으로 생성해준다. 기존에 Lombok `@Value`나 수십 줄짜리 보일러플레이트가 필요했던 DTO/값 객체를 한 줄로 표현할 수 있다.

```java
// 기존 방식
public final class Point {
    private final int x;
    private final int y;
    public Point(int x, int y) { this.x = x; this.y = y; }
    public int x() { return x; }
    public int y() { return y; }
    // equals, hashCode, toString 생략...
}

// record
public record Point(int x, int y) {}

// 사용
var p = new Point(3, 5);
System.out.println(p.x());   // 3
System.out.println(p);       // Point[x=3, y=5]
```

컴팩트 생성자(compact constructor)로 유효성 검증도 추가할 수 있다.

```java
public record Range(int min, int max) {
    Range {
        if (min > max) throw new IllegalArgumentException("min > max");
    }
}
```

> 💡 **왜 중요한가**: 불변 객체 설계를 강제하면서 코드량을 대폭 줄여주며, Java 21 `switch` 패턴 매칭과 결합하면 대수적 데이터 타입(ADT) 스타일의 안전한 도메인 모델링이 가능하다.

### `CompletableFuture`로 비동기 파이프라인 구성하기

`CompletableFuture`는 Java 8부터 제공된 비동기 작업 조합 API로, `thenApply`·`thenCompose`·`exceptionally` 등을 체이닝해 콜백 지옥 없이 비동기 흐름을 선언적으로 표현할 수 있다. 여러 Future를 병렬로 실행하고 결과를 합치는 `allOf`·`anyOf`도 지원한다.

```java
CompletableFuture<String> result = CompletableFuture
    .supplyAsync(() -> fetchUserId())          // 별도 스레드에서 실행
    .thenApply(id -> "user-" + id)             // 값 변환 (동기)
    .thenCompose(key -> fetchProfile(key))     // 반환값이 Future인 경우 flatten
    .exceptionally(ex -> "default-profile");   // 예외 복구

// 병렬 조합
CompletableFuture<Void> all = CompletableFuture.allOf(futureA, futureB, futureC);
all.thenRun(() -> System.out.println("모든 작업 완료"));
```

`thenApply`는 `map`, `thenCompose`는 `flatMap`에 해당한다고 이해하면 직관적이다.

> 💡 **왜 중요한가**: 스레드를 직접 관리하지 않고도 I/O 바운드 작업을 병렬로 조합할 수 있어, 외부 API 다중 호출이나 DB 조회 병렬화 같은 실무 패턴에서 코드 가독성과 성능을 동시에 확보할 수 있다.
