### 1. Virtual Threads로 고성능 동시성 서버 구현하기 (Java 21 정식)

`Thread.ofVirtual().start()` 또는 `Executors.newVirtualThreadPerTaskExecutor()`로 수백만 개의 가상 스레드를 OS 스레드 비용 없이 생성할 수 있습니다. 블로킹 I/O 호출 시 JVM이 자동으로 캐리어 스레드에서 언마운트하므로, 기존의 동기 스타일 코드를 그대로 유지하면서 리액티브 수준의 처리량을 달성할 수 있습니다. 단, `synchronized` 블록 내부에서 블로킹이 발생하면 캐리어 스레드가 고정(pinning)되므로 `ReentrantLock`으로 대체하는 것이 좋습니다.

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 10_000).forEach(i ->
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1));
            return fetchUserData(i);
        })
    );
}
```

> 💡 **왜 중요한가**: 기존 플랫폼 스레드 풀의 "스레드 수 = 동시성 한계" 제약을 제거해, 복잡한 비동기 프레임워크 없이도 동기 코드로 확장 가능한 서버를 작성할 수 있습니다.

### 2. `switch` 표현식과 패턴 매칭으로 분기 로직 간결하게 표현하기 (Java 21 정식)

Java 21부터 `switch`에서 타입 패턴, `when` 가드, `null` 케이스를 함께 사용할 수 있어 복잡한 if-else 체인을 선언적으로 대체할 수 있습니다. 화살표 문법(`->`)은 fall-through를 방지하고 값을 직접 반환하므로, 표현식으로 바로 변수에 할당하거나 반환할 수 있습니다. `sealed` 인터페이스와 결합하면 컴파일러가 모든 하위 타입을 검사하는 완전성(exhaustiveness) 검증까지 자동으로 수행합니다.

```java
String describe(Object obj) {
    return switch (obj) {
        case null          -> "없음";
        case Integer i when i < 0 -> "음수: " + i;
        case Integer i     -> "양수: " + i;
        case String s      -> "문자열 길이: " + s.length();
        case int[] arr     -> "배열 크기: " + arr.length;
        default            -> "알 수 없는 타입";
    };
}
```

> 💡 **왜 중요한가**: 타입 검사 → 캐스팅 → 분기로 이어지는 보일러플레이트를 제거하고, `sealed` 타입과 함께 쓰면 새 하위 타입 추가 시 누락된 케이스를 컴파일러가 강제로 잡아줍니다.
