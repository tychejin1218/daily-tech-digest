### 1. 부분 인덱스(Partial Index) — 조건부 인덱싱으로 저장 공간과 성능 동시 확보

부분 인덱스는 테이블 전체가 아닌 `WHERE` 조건을 만족하는 행에만 인덱스를 생성하는 기법입니다. 예를 들어 주문 테이블에서 `status = 'PENDING'`인 행이 전체의 5%에 불과하다면, `CREATE INDEX idx_pending ON orders(created_at) WHERE status = 'PENDING'`처럼 해당 조건의 행에만 인덱스를 걸 수 있습니다. 인덱스 크기가 대폭 줄어들어 메모리 적중률이 높아지고, INSERT/UPDATE 시 인덱스 유지 비용도 조건에 해당하는 행에만 발생하므로 쓰기 성능까지 개선됩니다.

> 💡 **왜 중요한가**: 대부분의 쿼리가 특정 조건의 소수 데이터만 조회하는 패턴에서, 풀 인덱스 대비 수십 배 작은 인덱스로 동일한 조회 성능을 얻을 수 있기 때문입니다.

### 2. CTE(Common Table Expression)와 재귀 쿼리 — 계층 구조 데이터를 SQL로 탐색하기

`WITH` 절로 정의하는 CTE는 복잡한 서브쿼리를 이름 붙여 분리함으로써 가독성을 높이고, 같은 결과 집합을 여러 번 참조할 때 중복 로직을 제거합니다. 특히 `WITH RECURSIVE`를 사용하면 조직도, 카테고리 트리, BOM(Bill of Materials) 같은 계층 구조 데이터를 별도 애플리케이션 로직 없이 단일 SQL로 탐색할 수 있습니다. 다만 재귀 깊이가 깊어지면 성능이 급락할 수 있으므로, `MAXRECURSION` 힌트나 깊이 제한 컬럼을 두어 무한 루프를 방지해야 합니다.

> 💡 **왜 중요한가**: 트리·그래프 형태의 데이터를 애플리케이션에서 반복 쿼리로 처리하면 N번의 DB 왕복이 발생하지만, 재귀 CTE 한 번이면 DB 내부에서 전체 경로를 완성할 수 있기 때문입니다.
