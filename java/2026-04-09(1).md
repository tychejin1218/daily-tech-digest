### Sealed Classes로 타입 계층 명시적으로 제한하기

`sealed` 키워드(Java 17)를 사용하면 어떤 클래스/인터페이스가 특정 타입을 상속할 수 있는지 컴파일 타임에 명시적으로 선언할 수 있다. `permits` 절에 허용된 서브타입만 `extends`/`implements`가 가능하며, 각 서브타입은 반드시 `final`, `sealed`, 또는 `non-sealed` 중 하나를 선언해야 한다. 이를 통해 컴파일러가 `switch` 패턴 매칭에서 모든 케이스가 처리됐는지 exhaustiveness 검사를 수행할 수 있게 된다.

```java
sealed interface Shape permits Circle, Rectangle, Triangle {}

record Circle(double radius) implements Shape {}
record Rectangle(double width, double height) implements Shape {}
final class Triangle implements Shape { /* ... */ }

double area(Shape s) {
    return switch (s) {
        case Circle c    -> Math.PI * c.radius() * c.radius();
        case Rectangle r -> r.width() * r.height();
        case Triangle t  -> /* ... */ 0;
        // default 없어도 컴파일 통과 — 컴파일러가 완전성을 보장
    };
}
```

> 💡 **왜 중요한가**: 도메인 모델의 유효한 상태를 타입 시스템으로 강제해 런타임 `ClassCastException`과 누락된 분기 버그를 컴파일 단계에서 차단한다.

### Stream Gatherers(Java 22+)로 커스텀 중간 연산 정의하기

`Stream.gather(Gatherer)` API(Java 22 Preview → Java 24 정식)는 기존 `map`, `filter`, `flatMap`으로는 표현하기 어려운 **상태 유지(stateful)** 중간 연산을 직접 구현할 수 있게 해준다. `Gatherer`는 초기 상태 공급자(initializer), 요소 통합자(integrator), 완결 처리자(finisher)로 구성되며, 슬라이딩 윈도우·중복 제거·배치 그룹핑 같은 패턴을 스트림 파이프라인 안에서 깔끔하게 표현할 수 있다. Java 24부터는 `Gatherers` 유틸리티 클래스에 `windowSliding`, `windowFixed`, `scan`, `fold` 등의 빌트인 구현도 제공된다.

```java
import java.util.stream.Gatherers;

// 슬라이딩 윈도우: [1,2,3,4,5] → [[1,2,3],[2,3,4],[3,4,5]]
List<List<Integer>> windows = Stream.of(1, 2, 3, 4, 5)
    .gather(Gatherers.windowSliding(3))
    .toList();

// 커스텀 Gatherer: 연속된 같은 값 제거
Gatherer<Integer, ?, Integer> dedupConsecutive = Gatherer.ofSequential(
    () -> new int[]{Integer.MIN_VALUE},
    (state, element, downstream) -> {
        if (state[0] != element) {
            state[0] = element;
            downstream.push(element);
        }
        return true;
    }
);
```

> 💡 **왜 중요한가**: 지금까지 `Collector`나 외부 루프로 우회하던 복잡한 시퀀스 변환을 스트림의 지연 평가(lazy evaluation)와 병렬 처리 모델 안에서 일관되게 표현할 수 있다.
