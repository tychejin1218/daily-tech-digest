### 1. Unnamed Variables and Patterns(`_`)로 사용하지 않는 변수와 패턴 컴포넌트 명시하기 (Java 22 정식)

`_` 기호를 사용해 의도적으로 무시하는 람다 매개변수, catch 절의 예외, try-with-resources 리소스, 그리고 레코드 패턴의 컴포넌트를 표시할 수 있습니다. 이름을 지어내거나 IDE의 "사용되지 않음" 경고를 억제할 필요 없이 "이 값은 의도적으로 버린다"는 사실을 컴파일러와 동료 개발자 모두에게 명확히 전달합니다.

```java
// 람다에서 사용하지 않는 인자
map.forEach((_, v) -> System.out.println(v));

// catch 절에서 예외 객체를 사용하지 않을 때
try {
    int n = Integer.parseInt(input);
} catch (NumberFormatException _) {
    return -1;
}

// 레코드 패턴에서 일부 컴포넌트만 추출
record Point(int x, int y) {}
if (obj instanceof Point(int x, _)) {
    System.out.println("x=" + x);
}

// 같은 블록에서 여러 번 사용 가능 (변수 충돌 없음)
int total = 0;
for (Order _ : orders) {
    total++;
}
```

> 💡 **왜 중요한가**: 코드 의도("이 값은 쓰지 않는다")를 문법으로 표현하면, 잘못된 변수 사용으로 인한 버그와 의미 없는 변수명(`ignored`, `unused`, `tmp`)을 동시에 줄일 수 있습니다.

### 2. Markdown Documentation Comments로 Javadoc을 더 읽기 쉽게 작성하기 (Java 23 정식)

JEP 467로 도입된 Markdown Javadoc은 `///`(슬래시 3개)로 시작하는 새로운 주석 형식이며, HTML 태그(`<p>`, `<ul>`, `<code>`) 대신 친숙한 Markdown 문법을 그대로 사용할 수 있습니다. 기존 `/** */` 스타일과 공존하므로 점진적으로 전환할 수 있고, IntelliJ·VS Code 등 최신 IDE는 미리보기와 링크를 자동으로 렌더링합니다.

```java
/// 사용자 정보를 조회한다.
///
/// 이 메서드는 다음 순서로 동작한다:
/// 1. 캐시에서 먼저 확인
/// 2. 없으면 DB 조회
/// 3. 결과를 캐시에 저장
///
/// 사용 예:
/// ```java
/// User user = service.findById(42L);
/// ```
///
/// 자세한 내용은 [User] 와 [#findAll()] 을 참고.
///
/// @param id 사용자 식별자 (1 이상)
/// @return 사용자 객체, 없으면 `null`
/// @throws IllegalArgumentException `id`가 0 이하인 경우
public User findById(long id) { ... }
```

링크는 `[User]`처럼 대괄호 안에 요소 참조를 넣으면 자동으로 `{@link}`로 변환되고, 코드 블록·리스트·강조 표시도 Markdown 그대로 작동합니다.

> 💡 **왜 중요한가**: HTML 태그로 가득한 기존 Javadoc은 작성·리뷰 모두 부담이 컸지만, Markdown 문법은 README나 GitHub PR과 동일한 감각으로 쓸 수 있어 문서화 비용을 크게 낮춥니다.
