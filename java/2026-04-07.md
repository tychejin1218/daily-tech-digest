### 1. Virtual Threads (Project Loom) — 블로킹 I/O의 패러다임 전환

Java 21에서 정식 출시된 Virtual Thread는 JVM이 내부적으로 수백만 개의 경량 스레드를 관리할 수 있게 해준다. 기존 플랫폼 스레드는 OS 스레드와 1:1 매핑되어 스레드 풀 크기에 제약이 있었지만, Virtual Thread는 블로킹 시 캐리어 스레드를 점유하지 않아 높은 동시성을 달성할 수 있다. `Executors.newVirtualThreadPerTaskExecutor()`로 간단히 적용 가능하다.

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 100_000).forEach(i ->
        executor.submit(() -> {
            Thread.sleep(Duration.ofMillis(100)); // 블로킹해도 OK
            return i;
        })
    );
}
```

> 💡 **왜 중요한가**: 기존 리액티브 프로그래밍(WebFlux 등)의 복잡성 없이 동기 코드 스타일을 유지하면서 높은 처리량을 달성할 수 있어, 서버 애플리케이션 설계를 근본적으로 단순화한다.

### 2. Sequenced Collections — 순서 보장 컬렉션 인터페이스

Java 21에서 도입된 `SequencedCollection` 인터페이스는 `List`, `Deque`, `LinkedHashSet` 등 순서가 있는 컬렉션에 통일된 API를 제공한다. 기존에는 첫/마지막 요소 접근 방식이 컬렉션마다 달라(`list.get(0)` vs `deque.peekFirst()`) 혼란이 많았다. 이제 `getFirst()`, `getLast()`, `reversed()` 메서드를 일관되게 사용할 수 있다.

```java
List<String> list = new ArrayList<>(List.of("a", "b", "c"));
list.getFirst(); // "a"
list.getLast();  // "c"

// 역순 뷰 (새 컬렉션 생성 아님)
for (String s : list.reversed()) {
    System.out.println(s); // c, b, a
}

LinkedHashSet<Integer> set = new LinkedHashSet<>(Set.of(1, 2, 3));
set.addFirst(0); // 맨 앞에 삽입
```

> 💡 **왜 중요한가**: 컬렉션 타입에 관계없이 동일한 메서드명으로 양 끝 요소를 다룰 수 있어 코드 일관성이 높아지고, 타입 변환 없이 다형적으로 처리할 수 있다.
