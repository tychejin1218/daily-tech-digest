### Spring Modulith로 모듈러 모놀리스 아키텍처 구축

Spring Modulith는 단일 Spring Boot 애플리케이션 내에서 비즈니스 모듈 간 경계를 명확히 하고 의존성을 검증해주는 프레임워크입니다. 패키지 단위로 모듈을 정의하고 `ApplicationModules.of(Application.class).verify()`로 순환 의존이나 허용되지 않은 모듈 간 참조를 테스트 시점에 잡아낼 수 있으며, 모듈 간 통신은 `ApplicationEventPublisher`를 통한 이벤트 기반으로 느슨하게 연결됩니다. `@ApplicationModuleTest`로 특정 모듈만 격리해 통합 테스트하는 것도 가능합니다.

> 💡 **왜 중요한가**: 마이크로서비스의 분산 복잡도 없이도 경계가 지켜지는 구조를 강제해, 추후 서비스 분리 시 이전 비용을 극적으로 낮춥니다.

### Spring Boot 3.3+ `RestClient`로 동기 HTTP 클라이언트 현대화

`RestClient`는 `RestTemplate`의 후속으로 도입된 동기 방식 HTTP 클라이언트로, `WebClient`와 유사한 플루언트 API를 제공하면서도 리액티브 의존성 없이 사용할 수 있습니다. `RestClient.builder().baseUrl(...).defaultHeader(...).requestInterceptor(...).build()` 형태로 구성하며, `.retrieve().body(Type.class)` 또는 `.toEntity(...)`로 응답을 받고 `.onStatus(...)`로 상태 코드별 예외 처리를 선언할 수 있습니다. 기존 `RestTemplate` 코드를 대체하면서도 테스트 시 `MockRestServiceServer`를 그대로 재사용할 수 있어 마이그레이션 부담이 적습니다.

> 💡 **왜 중요한가**: `RestTemplate`은 유지보수 모드로 들어갔고 `WebClient`는 블로킹 환경에 과한 선택이라, 동기 API 호출의 기본값은 이제 `RestClient`가 되어야 합니다.
