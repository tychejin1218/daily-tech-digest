### 1. Virtual Threads (Project Loom)로 고성능 동시성 구현하기

Java 21에서 정식 도입된 Virtual Threads는 OS 스레드 대신 JVM이 직접 관리하는 경량 스레드로, 수백만 개를 동시에 생성해도 메모리 부담이 적다. 기존 `ExecutorService`와 동일한 API를 사용하면서도 블로킹 I/O 작업에서 플랫폼 스레드를 점유하지 않아 처리량이 획기적으로 향상된다.

```java
// Virtual Thread 기반 ExecutorService 생성
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 100_000).forEach(i ->
        executor.submit(() -> {
            // 블로킹 I/O도 OK — 플랫폼 스레드를 점유하지 않음
            Thread.sleep(Duration.ofMillis(100));
            return i;
        })
    );
} // try-with-resources로 자동 shutdown
```

> 💡 **왜 중요한가**: 스레드 풀 크기 튜닝 없이도 I/O 바운드 애플리케이션의 동시 처리량을 수십 배 높일 수 있어, 리액티브 프로그래밍 없이도 높은 확장성을 달성할 수 있다.

### 2. `SequencedCollection` 인터페이스로 순서 있는 컬렉션 통합 처리

Java 21에서 도입된 `SequencedCollection`은 `List`, `Deque`, `LinkedHashSet` 등 순서를 가진 컬렉션의 공통 인터페이스로, `getFirst()`, `getLast()`, `reversed()` 메서드를 표준화했다. 이전에는 컬렉션 타입마다 첫/마지막 요소 접근 방법이 달라 혼란스러웠지만, 이제 단일 인터페이스로 일관되게 처리할 수 있다.

```java
List<String> list = new ArrayList<>(List.of("A", "B", "C"));

// 타입에 관계없이 동일한 API 사용
System.out.println(list.getFirst()); // "A"
System.out.println(list.getLast());  // "C"

// 역순 뷰 (새 컬렉션 복사 없이 뷰 반환)
list.reversed().forEach(System.out::println); // C, B, A

// Map도 SequencedMap으로 지원
SequencedMap<String, Integer> map = new LinkedHashMap<>();
map.put("one", 1); map.put("two", 2);
System.out.println(map.firstEntry()); // one=1
```

> 💡 **왜 중요한가**: 컬렉션 타입마다 달랐던 순서 접근 API가 통일되어 코드 가독성이 높아지고, 제네릭 메서드 작성 시 불필요한 타입 분기를 제거할 수 있다.
