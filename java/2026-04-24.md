### 1. Stream Gatherers로 커스텀 중간 연산(Intermediate Operation) 만들기 (Java 24 정식)

`Stream.gather(Gatherer)`는 `map`/`filter`/`flatMap`으로 표현하기 어려웠던 상태 기반 변환(슬라이딩 윈도우, 연속 중복 제거, N개씩 묶기 등)을 직접 구현할 수 있는 공식 확장 지점입니다. `Gatherer.ofSequential()`에 초기 상태, 원소별 integrator, 종료 finisher를 조합하여 선언적이면서도 병렬/순차 모드를 모두 지원하는 파이프라인을 만들 수 있습니다.

```java
// 연속된 3개씩 슬라이딩 윈도우
List<List<Integer>> windows = Stream.of(1, 2, 3, 4, 5)
    .gather(Gatherers.windowSliding(3))
    .toList();
// [[1,2,3], [2,3,4], [3,4,5]]
```

> 💡 **왜 중요한가**: 그동안 `reduce`/`collect` 트릭이나 외부 라이브러리(예: StreamEx)로만 가능했던 상태 기반 스트림 연산이 표준 API로 들어와, 도메인 로직을 훨씬 자연스럽게 스트림 체인에 녹일 수 있습니다.

### 2. Sequenced Collections로 순서 있는 컬렉션의 양 끝을 일관된 API로 다루기 (Java 21 정식)

`SequencedCollection`, `SequencedSet`, `SequencedMap` 인터페이스가 도입되어 `List`, `Deque`, `LinkedHashSet/Map`, `TreeSet/Map` 등 "순서"가 있는 자료구조에서 `getFirst()`, `getLast()`, `addFirst()`, `addLast()`, `reversed()`를 동일한 이름으로 쓸 수 있게 되었습니다. 예전에는 `list.get(0)`, `deque.peekFirst()`, `treeSet.first()`처럼 구현체마다 제각각이던 API가 통일된 셈입니다.

```java
LinkedHashSet<String> set = new LinkedHashSet<>(List.of("a", "b", "c"));
String first = set.getFirst();       // "a"  (이전에는 불가능)
String last  = set.getLast();        // "c"
var reversed = set.reversed();       // View, 원본과 공유
```

> 💡 **왜 중요한가**: 코드 리뷰에서 자주 지적되던 "이 컬렉션의 첫/마지막 원소를 어떻게 꺼내지?" 문제를 타입 시스템 수준에서 해소해, 구현체 교체 시에도 호출 코드가 그대로 유지됩니다.
