# Java - 2026-04-08

### Virtual Thread (가상 스레드)의 핀닝(Pinning) 문제

Java 21에서 정식 도입된 가상 스레드는 `synchronized` 블록 내부 또는 네이티브 메서드 호출 시 캐리어 스레드에 **핀닝(pinning)**되어 블로킹이 발생할 수 있다. 이 상태에서는 가상 스레드의 핵심 장점인 스레드 마운트/언마운트가 동작하지 않아 성능 이점이 사라진다. `synchronized` 대신 `ReentrantLock`을 사용하면 핀닝을 피할 수 있으며, JVM 옵션 `-Djdk.tracePinnedThreads=full`로 핀닝 발생 지점을 추적할 수 있다.

> 💡 **왜 중요한가**: 가상 스레드를 도입했음에도 핀닝으로 인해 성능 개선이 없는 경우가 발생할 수 있어, 마이그레이션 시 반드시 확인해야 한다.

### `SequencedCollection` — Java 21의 새로운 컬렉션 인터페이스

Java 21은 순서가 보장된 컬렉션을 위한 `SequencedCollection` 인터페이스를 도입했다. `getFirst()`, `getLast()`, `addFirst()`, `addLast()`, `reversed()` 등의 메서드를 통해 `List`, `Deque`, `LinkedHashSet` 등에서 일관된 방식으로 첫/마지막 원소에 접근할 수 있다. 기존에는 자료구조마다 `get(0)`, `peek()`, `iterator().next()` 등 제각각의 방법을 사용해야 했던 불편함이 해소되었다.

> 💡 **왜 중요한가**: 순서 있는 컬렉션 조작 코드의 가독성과 일관성을 높여 실수를 줄일 수 있다.
