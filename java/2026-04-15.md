### 1. Implicitly Declared Classes와 간소화된 `main()` 메서드로 Java 진입점 단순화하기 (Java 24 정식)

Java 24부터 클래스 선언 없이 `void main()` 만으로 프로그램을 시작할 수 있습니다. 기존의 `public static void main(String[] args)` 시그니처와 클래스 보일러플레이트가 필요 없어져, 스크립트성 코드나 학습 목적의 코드를 훨씬 간결하게 작성할 수 있습니다. `static`이 아닌 인스턴스 메서드로도 동작하므로 인스턴스 필드와 메서드를 자연스럽게 활용할 수 있습니다.

```java
// Main.java — 클래스 선언 없이 바로 실행 가능
void main() {
    var greet = "Hello, Java 24!";
    println(greet);  // System.out.println 대신 간소화된 io 메서드 사용
}
```

> 💡 **왜 중요한가**: Java의 가장 큰 진입 장벽이었던 보일러플레이트를 제거하여, Python이나 Kotlin처럼 빠른 프로토타이핑과 교육 현장에서의 활용성이 크게 향상됩니다.

### 2. `Stream.mapMulti()`로 flatMap보다 효율적인 일대다 변환 처리하기 (Java 16+)

`mapMulti()`는 `flatMap()`의 명령형 대안으로, 중간 스트림 객체를 생성하지 않고 `Consumer`를 통해 결과를 직접 푸시하는 방식입니다. 조건부로 0개, 1개, 또는 여러 개의 요소를 유연하게 내보낼 수 있어, 특히 변환 결과가 소수이거나 조건 분기가 복잡한 경우 `flatMap()`보다 성능과 가독성 모두에서 유리합니다.

```java
// flatMap: 매번 Stream 객체 생성
list.stream()
    .flatMap(order -> order.getItems().stream())
    .toList();

// mapMulti: Consumer로 직접 푸시 — 중간 Stream 생성 비용 없음
list.stream()
    .<Item>mapMulti((order, consumer) -> {
        for (Item item : order.getItems()) {
            if (item.isActive()) {
                consumer.accept(item);
            }
        }
    })
    .toList();
```

> 💡 **왜 중요한가**: 각 요소마다 중간 `Stream` 객체를 할당하는 `flatMap`의 GC 부담을 제거하면서도, 필터링과 변환을 하나의 람다 안에서 명령형으로 자유롭게 제어할 수 있습니다.
