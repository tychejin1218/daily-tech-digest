### 1. `HttpClient`의 비동기 API로 논블로킹 HTTP 통신 구현하기 (Java 11 정식)

Java 11부터 표준 라이브러리에 포함된 `java.net.http.HttpClient`는 `sendAsync()` 메서드를 통해 `CompletableFuture<HttpResponse<T>>`를 반환하므로 별도의 외부 라이브러리 없이 논블로킹 방식으로 여러 HTTP 요청을 동시에 처리할 수 있습니다. `BodyHandlers.ofString()`, `ofByteArray()`, `ofFile()` 등 다양한 응답 처리기를 조합하고, `thenApply()`, `thenCompose()`로 후속 처리를 체이닝하면 콜백 지옥 없이 깔끔한 파이프라인을 구성할 수 있습니다. HTTP/2와 WebSocket도 기본 지원하여 현대적인 통신 요구사항을 충족합니다.

```java
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)
    .connectTimeout(Duration.ofSeconds(10))
    .build();

List<CompletableFuture<String>> futures = urls.stream()
    .map(url -> HttpRequest.newBuilder(URI.create(url)).build())
    .map(req -> client.sendAsync(req, BodyHandlers.ofString())
                      .thenApply(HttpResponse::body))
    .toList();

CompletableFuture.allOf(futures.toArray(CompletableFuture[]::new))
    .thenRun(() -> System.out.println("모든 요청 완료"));
```

> 💡 **왜 중요한가**: 외부 의존성(Apache HttpClient, OkHttp) 없이 표준 API만으로 고성능 비동기 HTTP 클라이언트를 구축할 수 있어 애플리케이션의 경량화와 유지보수성을 동시에 확보할 수 있습니다.

### 2. `VarHandle`로 원자적 필드 연산과 메모리 배리어 세밀하게 제어하기 (Java 9+)

`VarHandle`은 `sun.misc.Unsafe`의 안전한 대체제로, 필드나 배열 요소에 대한 저수준 원자 연산과 메모리 순서(memory ordering)를 타입 안전하게 제어할 수 있게 해줍니다. `getAcquire`/`setRelease`, `getVolatile`/`setVolatile`, `compareAndSet` 등 다양한 접근 모드를 제공하여 `AtomicInteger` 같은 래퍼 클래스를 감싸지 않고도 일반 필드를 원자적으로 다룰 수 있어 메모리 오버헤드를 줄일 수 있습니다. 특히 락프리(lock-free) 자료구조를 직접 구현하거나, 고성능 동시성 라이브러리를 만들 때 필수적인 도구입니다.

```java
public class Counter {
    private static final VarHandle VALUE;
    private volatile int value;

    static {
        try {
            VALUE = MethodHandles.lookup()
                .findVarHandle(Counter.class, "value", int.class);
        } catch (ReflectiveOperationException e) {
            throw new ExceptionInInitializerError(e);
        }
    }

    public int incrementAndGet() {
        int current;
        do {
            current = (int) VALUE.getAcquire(this);
        } while (!VALUE.compareAndSet(this, current, current + 1));
        return current + 1;
    }
}
```

> 💡 **왜 중요한가**: `Unsafe`에 의존하지 않고도 JIT 컴파일러가 최적화 가능한 형태로 저수준 동시성 제어를 구현할 수 있어, 고성능 라이브러리 개발자에게 안전한 표준 경로를 제공합니다.
