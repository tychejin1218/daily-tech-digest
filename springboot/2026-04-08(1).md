# Spring Boot - 2026-04-08

### 1. `@Transactional` 전파 속성(Propagation) 이해와 함정 피하기

`@Transactional`은 기본적으로 `REQUIRED` 전파 속성을 사용하여 기존 트랜잭션이 있으면 합류하고, 없으면 새로 시작한다. 주의할 점은 **같은 클래스 내에서 `@Transactional` 메서드를 호출하면 프록시를 거치지 않아 트랜잭션이 적용되지 않는** 자기 호출(self-invocation) 문제다. 이를 피하려면 메서드를 별도 빈으로 분리하거나 `REQUIRES_NEW`로 새 트랜잭션이 필요한 경우 외부 빈으로 위임해야 한다.

```java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final PaymentService paymentService;

    @Transactional
    public void placeOrder(Order order) {
        saveOrder(order);
        // 결제는 반드시 별도 트랜잭션 — 실패해도 주문 저장은 롤백하지 않으려면
        paymentService.charge(order);  // 외부 빈 호출 → 프록시 통과 O
    }

    private void saveOrder(Order order) { /* ... */ }
}

@Service
public class PaymentService {

    // REQUIRES_NEW: 부모 트랜잭션과 독립적으로 커밋/롤백
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void charge(Order order) {
        // 결제 처리
    }
}
```

> 💡 **왜 중요한가**: 자기 호출 함정과 전파 속성 오용은 데이터 정합성 버그로 이어지며, 테스트에서는 정상 동작해도 운영에서 트랜잭션 경계가 어긋나는 장애를 유발한다.

### 2. `@ConfigurationProperties`로 타입 안전한 설정 관리

`@Value`로 설정값을 하나씩 주입하면 오타 감지가 어렵고 관련 설정이 코드 전역에 흩어진다. `@ConfigurationProperties`를 사용하면 `application.yml`의 설정 그룹을 **타입이 검증된 POJO**로 바인딩할 수 있어 IDE 자동완성과 컴파일 타임 오류 감지가 가능하다. `spring-boot-configuration-processor` 의존성을 추가하면 메타데이터도 자동 생성된다.

```yaml
# application.yml
app:
  mail:
    host: smtp.example.com
    port: 587
    username: noreply@example.com
    timeout: 5000
```

```java
@ConfigurationProperties(prefix = "app.mail")
@Validated
public record MailProperties(
    @NotBlank String host,
    @Min(1) @Max(65535) int port,
    @NotBlank String username,
    @Positive long timeout
) {}

// @EnableConfigurationProperties 또는 @ConfigurationPropertiesScan 로 등록
@SpringBootApplication
@ConfigurationPropertiesScan
public class Application { ... }

@Service
@RequiredArgsConstructor
public class MailService {

    private final MailProperties mailProperties;

    public void send(String to, String body) {
        // mailProperties.host(), mailProperties.port() 등 타입 안전하게 접근
    }
}
```

> 💡 **왜 중요한가**: 설정값은 배포 환경마다 달라지는 핵심 외부 변수이므로, 타입 안전성과 검증을 갖추면 잘못된 설정으로 인한 런타임 장애를 사전에 차단할 수 있다.
