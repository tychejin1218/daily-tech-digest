### 1. Foreign Function & Memory API로 JNI 없이 네이티브 코드 호출하기 (Java 22 정식)

Java 22에서 정식 확정된 FFM API(`java.lang.foreign`)를 사용하면 복잡한 JNI 코드 없이도 C/C++ 같은 네이티브 라이브러리를 직접 호출할 수 있습니다. `Arena`를 통해 off-heap 메모리를 안전하게 할당·해제하고, `Linker`와 `FunctionDescriptor`로 네이티브 함수 시그니처를 선언적으로 기술합니다. `MemorySegment` 기반의 메모리 접근은 범위 검사가 내장되어 있어 기존 `Unsafe` 사용 대비 안전성이 크게 향상됩니다.

```java
// C 표준 라이브러리의 strlen 호출 예시
try (Arena arena = Arena.ofConfined()) {
    MemorySegment cString = arena.allocateFrom("Hello, Panama!");
    
    Linker linker = Linker.nativeLinker();
    MethodHandle strlen = linker.downcallHandle(
        linker.defaultLookup().find("strlen").orElseThrow(),
        FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS)
    );
    
    long len = (long) strlen.invoke(cString);  // 결과: 14
}
```

> 💡 **왜 중요한가**: JNI의 boilerplate와 메모리 안전 문제를 해결하며, 고성능 네이티브 연동이 필요한 시스템 프로그래밍을 Java에서도 실용적으로 수행할 수 있게 해줍니다.

### 2. Flexible Constructor Bodies로 `super()` 호출 전 검증 로직 작성하기 (Java 24)

Java 24에서 정식 도입된 Flexible Constructor Bodies는 생성자 내에서 `super()` 또는 `this()` 호출 **이전에** 인자 검증, 변환 로직을 작성할 수 있게 해줍니다. 기존에는 `super()`가 반드시 생성자의 첫 문장이어야 했기 때문에, 검증 로직을 별도 `static` 팩토리 메서드나 헬퍼로 우회해야 했습니다. 이제는 `this`의 필드에 접근하지 않는 범위 내에서 자유롭게 선행 코드를 배치할 수 있습니다.

```java
public class PositiveRange extends Range {
    public PositiveRange(int low, int high) {
        // super() 호출 전에 검증 가능 (Java 24)
        if (low < 0 || high < 0) {
            throw new IllegalArgumentException("값은 양수여야 합니다: " + low + ", " + high);
        }
        int normalizedLow = Math.min(low, high);
        int normalizedHigh = Math.max(low, high);
        
        super(normalizedLow, normalizedHigh);  // 검증·변환 후 호출
    }
}
```

> 💡 **왜 중요한가**: 생성자 인자 검증을 위해 `static` 헬퍼 메서드를 만드는 관용적 우회 패턴이 불필요해지며, 객체 생성 의도를 생성자 한 곳에 명확하게 표현할 수 있습니다.
