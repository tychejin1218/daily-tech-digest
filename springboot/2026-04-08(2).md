### `@ConfigurationProperties`로 타입 안전한 설정 관리

`@Value`로 프로퍼티를 하나씩 주입하면 오타나 타입 불일치가 런타임에야 드러난다. `@ConfigurationProperties`를 사용하면 `application.yml` 의 설정 그룹을 POJO에 바인딩하여 **컴파일 타임 타입 검사**와 IDE 자동완성, `@Validated`를 통한 제약 조건 검증까지 한 번에 얻을 수 있다.

```java
// application.yml
// app:
//   jwt:
//     secret: my-secret-key
//     expiration-ms: 3600000

@Validated
@ConfigurationProperties(prefix = "app.jwt")
public record JwtProperties(
    @NotBlank String secret,
    @Positive long expirationMs
) {}

// 사용 측 - 빈으로 주입받아 사용
@Service
public class JwtService {

    private final JwtProperties jwtProperties;

    public JwtService(JwtProperties jwtProperties) {
        this.jwtProperties = jwtProperties;
    }

    public String generateToken(String subject) {
        return Jwts.builder()
            .subject(subject)
            .expiration(new Date(System.currentTimeMillis() + jwtProperties.expirationMs()))
            .signWith(Keys.hmacShaKeyFor(jwtProperties.secret().getBytes()))
            .compact();
    }
}
```

Spring Boot 2.2 이상에서는 `@EnableConfigurationProperties` 없이도 `@ConfigurationPropertiesScan`이나 `@SpringBootApplication`의 스캔으로 자동 등록된다. `spring-boot-configuration-processor` 의존성을 추가하면 IDE에서 `.yml` 작성 시 자동완성도 지원된다.

> 💡 **왜 중요한가**: 설정값을 타입으로 묶어 관리하면 환경별 설정 오류를 애플리케이션 기동 시점에 즉시 감지할 수 있어 운영 장애를 예방한다.

### Spring AOP로 메서드 실행 시간 로깅 횡단 관심사 분리

서비스 레이어 곳곳에 `System.currentTimeMillis()` 코드를 심으면 비즈니스 로직이 오염된다. `@Around` 어드바이스를 사용하면 **단 하나의 Aspect 클래스**로 대상 메서드를 감싸 실행 시간을 측정하고, 비즈니스 코드는 전혀 건드리지 않아도 된다.

```java
@Aspect
@Component
@Slf4j
public class ExecutionTimeAspect {

    // @Service가 붙은 모든 public 메서드에 적용
    @Around("execution(public * (@org.springframework.stereotype.Service *).*(..))")
    public Object logExecutionTime(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        try {
            return pjp.proceed();
        } finally {
            long elapsed = System.currentTimeMillis() - start;
            log.info("[{}] {}ms",
                pjp.getSignature().toShortString(), elapsed);
        }
    }
}
```

포인트컷을 커스텀 어노테이션(`@Timed`)으로 정의하면 적용 범위를 더 세밀하게 제어할 수 있다. AOP는 Spring 빈에만 적용되므로 `new`로 직접 생성한 객체나 `private` 메서드는 프록시를 통하지 않아 어드바이스가 적용되지 않는다.

> 💡 **왜 중요한가**: 로깅·트랜잭션·보안 같은 횡단 관심사를 AOP로 분리하면 코드 중복이 사라지고, 핵심 비즈니스 로직의 가독성과 테스트 용이성이 모두 높아진다.
