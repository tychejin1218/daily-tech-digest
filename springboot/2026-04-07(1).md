### 1. `@Async`와 스레드 풀 설정으로 비동기 처리 최적화

`@Async`를 사용하면 메서드를 별도 스레드에서 비동기로 실행할 수 있지만, 기본 스레드 풀은 `SimpleAsyncTaskExecutor`로 **스레드를 무한 생성**하여 운영 환경에서 자원 고갈을 유발할 수 있다. `ThreadPoolTaskExecutor`를 직접 빈으로 등록하고 `corePoolSize`, `maxPoolSize`, `queueCapacity`를 명시적으로 설정해야 안정적인 비동기 처리가 가능하다.

```java
@Configuration
@EnableAsync
public class AsyncConfig {

    @Bean(name = "taskExecutor")
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(20);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("async-");
        executor.initialize();
        return executor;
    }
}

@Service
public class NotificationService {

    @Async("taskExecutor")
    public CompletableFuture<Void> sendEmail(String to) {
        // 비동기 이메일 발송
        return CompletableFuture.completedFuture(null);
    }
}
```

> 💡 **왜 중요한가**: 기본 Executor를 그대로 쓰면 트래픽 급증 시 스레드 폭발로 OOM이 발생할 수 있으므로, 스레드 풀 튜닝은 운영 안정성의 핵심이다.

### 2. `@SpringBootTest` 대신 슬라이스 테스트로 테스트 속도 개선

`@SpringBootTest`는 전체 애플리케이션 컨텍스트를 로드하여 테스트 속도가 매우 느려진다. Spring Boot는 레이어별 **슬라이스 테스트 어노테이션**(`@WebMvcTest`, `@DataJpaTest`, `@JsonTest`)을 제공하여 필요한 빈만 로드하므로 테스트 실행 속도를 수십 배 단축할 수 있다.

```java
// Controller 계층만 테스트 - JPA, Service 빈 로드 안 함
@WebMvcTest(UserController.class)
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private UserService userService;

    @Test
    void getUser_returnsOk() throws Exception {
        given(userService.findById(1L)).willReturn(new UserDto(1L, "Alice"));

        mockMvc.perform(get("/users/1"))
               .andExpect(status().isOk())
               .andExpect(jsonPath("$.name").value("Alice"));
    }
}

// JPA 계층만 테스트 - 인메모리 DB 사용
@DataJpaTest
class UserRepositoryTest {

    @Autowired
    private UserRepository userRepository;

    @Test
    void save_and_findById() {
        User saved = userRepository.save(new User("Bob"));
        assertThat(userRepository.findById(saved.getId())).isPresent();
    }
}
```

> 💡 **왜 중요한가**: CI 파이프라인에서 전체 컨텍스트 로딩을 반복하면 빌드 시간이 기하급수적으로 늘어나므로, 슬라이스 테스트 습관화가 개발 생산성을 좌우한다.
